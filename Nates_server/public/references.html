This page lists and describes third-party APIs, libraries, and code that we used in our project.

APIs

Particle Device Cloud API

The Particle Device Cloud API is a REST API that allows for programmatic access to Particle devices. The API is used in our project in the following ways:

  • To validate the device ID by making an API call to the Particle Cloud to get the device information.
  • Retrieve current device status (i.e., online/offline) from the Particle Cloud.
  • To get the device name based on the product ID.
  • To request a new access token from the Particle Cloud.
  • To send parameter update requests to our Particle device using cloud functions.

Google Fonts API

The Google Fonts API provides a vast collection of fonts which we have implemented throughout our website. By utilizing this API, we've ensured a consistent and visually pleasing design across all text, headings, and paragraphs, enhancing the user experience with typography that is both readable and aesthetically pleasing.

We have also used the Node.js packages and frameworks breifly described below to develop our backend. This is by no means an exhastive list.

  • axios: Axios is a Promise-based HTTP client for JavaScript environments (such as browsers and Node.js). It allows developers to make asynchronous HTTP requests, handling features like request and response interception, and providing a clean, intuitive API.
  • bcryptjs: Bcryptjs is a JavaScript library used for securely hashing passwords. It implements the bcrypt hashing algorithm, which is designed to hash passwords in a secure manner, making it more computationally expensive for attackers to crack hashed passwords.
  • cookie-parser: Cookie-parser is an Express middleware used in Node.js applications to parse HTTP request cookies. It simplifies the handling of cookies by parsing them into a JavaScript object, allowing easier manipulation and access to cookie data.
  • debug: Debug is a debugging utility for Node.js and browsers that allows developers to add customizable logging messages throughout their code. It aids in debugging by providing detailed information about the program's execution flow and state.
  • express: Express is a widely used web application framework for Node.js. It simplifies the creation of web applications by providing a robust set of features for routing, handling HTTP requests and responses, middleware support, and creating APIs.
  • http-errors: Http-errors is a utility library that simplifies the creation and management of HTTP errors in Node.js applications. It allows developers to generate HTTP error objects with specific status codes and error messages, making error handling more structured.
  • jwt-simple: Jwt-simple is a JavaScript library used for encoding and decoding JSON Web Tokens (JWT) in Node.js applications. It enables the creation and verification of JWTs, often employed in implementing secure authentication and authorization mechanisms.
  • moment-timezone: Moment-timezone is a JavaScript library for working with dates, times, and time zones. It extends the capabilities of the Moment.js library by providing functionality to manage and convert time between different time zones.
  • mongoose: Mongoose is an Object Data Modeling (ODM) library for MongoDB and Node.js. It simplifies interactions with MongoDB databases by providing a schema-based solution for modeling application data and handling database operations.
  • morgan: Morgan is an HTTP request logging middleware for Node.js used with Express. It logs detailed information about incoming HTTP requests, such as request method, URL, status code, response size, and execution time, aiding in application monitoring and debugging.
  • particle-api-js: Particle-api-js is a JavaScript library used to interact with Particle IoT (Internet of Things) devices. It provides a convenient API to manage and control Particle devices, facilitating the development of IoT applications.
  • pug: Pug, formerly known as Jade, is a template engine for Node.js. It simplifies the process of writing HTML by using a concise and expressive syntax for generating HTML templates.
  • yargs: Yargs is a Node.js library used for parsing command-line arguments. It simplifies the process of handling command-line inputs by providing a flexible and easy-to-use API for defining and parsing command-line options and arguments.
